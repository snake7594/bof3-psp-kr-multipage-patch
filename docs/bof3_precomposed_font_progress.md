# BOF3 완성형 한글 폰트 진행 기록

## 방향 전환

초기에는 한글을 조합형으로 렌더링하는 방식을 사용했습니다. 하지만 표시 안정성과 예외 처리 문제가 많아, 가능한 한 완성형 한글 글리프를 직접 그려 넣는 방향으로 전환했습니다.

## 441자 안정안

- 게임의 기존 한자 렌더러를 그대로 사용했습니다.
- `0x12`, `0x13` 한자 토큰 영역의 441칸에 완성형 한글을 배치했습니다.
- EBOOT의 조합형 핸들러는 사용하지 않았습니다.
- 이 방식은 안정적이지만 한글 수가 부족했습니다.

## 441자 기준 커버리지

- 원 번역 기준 고유 한글: 1,147자
- 441자 선택 시 초과 한글: 706자
- 초과 출현 수: 6,855회
- 줄임 번역 기준 초과 출현 수: 4,543회

## 기호/가나 영역 활용 검토

- 현재 구조에서 실용적인 상한은 약 609자에서 656자입니다.
- 656자까지 늘려도 모든 번역을 무손실 수용할 수는 없습니다.
- 따라서 추가 폰트 페이지 접근 실험이 필요했습니다.

## 멀티페이지 실험

- `ENDKANJI.EMI`를 확장해 3개 441칸 페이지를 사용했습니다.
- 총 1,323칸을 확보했습니다.
- 현재 번역의 한글 1,147자를 전부 배치했습니다.
- 정적 인코딩 검증과 주입 검증은 통과했습니다.

## 남은 작업

실제 PPSSPP/PSP에서 추가 텍스처 페이지가 정상 표시되는지 확인해야 합니다.
